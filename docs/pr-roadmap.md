# PR Roadmap (Minimum)

- PR 1: bootstrap + ci + lint + test harness (`feat/service-core-layout`)
- PR 2: service core + config + logging (`feat/service-core-layout`)
- PR 3: auth + rate limiting + error model (`feat/auth-and-rate-limiting`)
- PR 4: telemetry + dashboards wiring (`feat/otel-and-metrics`)
- PR 5: operations control panel UX (`feat/ops-dashboard-ui`)
- PR 6: final docs + demo script + release prep (`docs/final-polish`)

Required in each PR:
- business context
- technical details
- screenshots (if UI)
- test strategy
- risks + rollback
- linked issue(s)
- one tradeoff discussion
